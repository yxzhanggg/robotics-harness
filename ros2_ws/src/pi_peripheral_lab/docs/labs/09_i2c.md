# 09 I2C

## Objective / Goal

Probe I2C devices from atlas and inspect SDA/SCL open-drain bus behavior.

## Learning Points

START, STOP, ACK, NACK, open-drain signaling, pull-up resistors, multi-drop bus,
and address conflict.

## BOM

atlas, ADS1115/MCP4725/OLED/IMU/BME280 module, oscilloscope, jumper wires.

## Circuit Diagram

```text
atlas SDA pin 3 ---- I2C SDA ---- pull-up to 3.3 V
atlas SCL pin 5 ---- I2C SCL ---- pull-up to 3.3 V
atlas GND pin 6 ---- module GND ---- scope ground
```

## Wiring Table

| Signal | atlas pin |
| --- | --- |
| SDA | 3 GPIO2 |
| SCL | 5 GPIO3 |
| GND | 6 |

## Oscilloscope Setup

Probe SDA and SCL. Use a trigger around START if available. Confirm idle high
and pulled-low pulses.

## Function Generator Setup

Not used.

## Run Commands

```bash
ros2 launch pi_peripheral_lab atlas_i2c_probe.launch.py dry_run:=false
ros2 topic echo /pin_lab/i2c/probe
```

## Expected Waveform

SDA/SCL idle high. Master pulls lines low for START, address, data, ACK/NACK,
and STOP. Found addresses appear in `/pin_lab/i2c/probe`.

## Common Mistakes

- Missing pull-ups or duplicate too-strong pull-ups.
- Wrong address.
- Powering module at 5 V without level shifting.
- I2C bus not enabled or permissions missing.

## Safety Checklist

- SDA/SCL are 3.3 V compatible.
- Pull-ups go to 3.3 V, not 5 V.
- Ground is common.

## HIL Pass/Fail Criteria

Pass: expected device address is listed and scope shows valid open-drain bus
activity.

Fail: no ACK, stuck-low line, or 5 V pull-up detected.

## Engineering Reuse

Teaching experiment: I2C probe node.

Reusable module: I2C address validation and adapter pattern.
